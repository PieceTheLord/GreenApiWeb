# Тестовое задание на должность «Девопс разработчик» в GREEN-API

Данный репозиторий содержит реализацию тестового задания по разработке интерфейса для взаимодействия с методами [GREEN-API](https://green-api.com/) и настройке автоматизированного процесса CI/CD.

## 🛠 Технологический стек

*   **Frontend:** Vanilla JavaScript (Fetch API), HTML5, CSS3.
*   **Infrastructure:** VPS (Ubuntu 24.04 LTS) на хостинге **Beget**.
*   **Containerization:** Docker & Docker Compose V2.
*   **Web Server:** Nginx (Alpine) с поддержкой **SSL/TLS**.
*   **CI/CD:** GitHub Actions (автоматизированный Pipeline).
*   **Security:** Let's Encrypt (Certbot) для управления SSL-сертификатами.
*   **Registry:** GitHub Container Registry (GHCR).

## 🏗 Архитектура и DevOps-решения

В ходе выполнения задания была спроектирована и внедрена отказоустойчивая инфраструктура:

### 1. Безопасность и HTTPS
*   Реализована поддержка **SSL/TLS**. Сертификаты генерируются с помощью **Certbot** (Let's Encrypt).
*   Конфигурация Nginx настроена на принудительный редирект (301 Move Permanently) с порта 80 на порт 443.
*   Использовано монтирование томов (**Docker Volumes**) для проброса сертификатов с хост-системы внутрь изолированного контейнера Nginx.

### 2. Оптимизация и производительность
*   **Nginx Tuning:** Увеличены буферы заголовков (`large_client_header_buffers 4 32k`), что предотвращает ошибку *400 Bad Request* при передаче больших Cookie-файлов (типичная проблема при разработке на localhost).
*   **Resource Management:** На слабом VPS настроен файл подкачки (**SWAP 2GB**) для предотвращения падения процессов из-за нехватки оперативной памяти (OOM).

### 3. CI/CD Workflow
Пайплайн в GitHub Actions полностью автоматизирует жизненный цикл приложения:
1.  **Build:** Сборка Docker-образа и тегирование.
2.  **Push:** Доставка образа в приватный реестр GHCR.
3.  **Config Sync:** Синхронизация `docker-compose.yml` с сервером через SCP.
4.  **Deploy:** Удаленное выполнение скриптов через SSH (pull обновленного образа, перезапуск сервисов, очистка старых образов для экономии дискового пространства).

## 📦 Инструкция по локальному запуску

Для локальной отладки рекомендуется использовать упрощенную конфигурацию без SSL (по умолчанию):

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/PieceTheLord/ВАШ_РЕПО.git
    cd ВАШ_РЕПО
    ```
2.  Запустите проект:
    ```bash
    docker compose up -d
    ```
3.  Доступ в браузере: `http://localhost`
