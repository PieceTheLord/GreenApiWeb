*   **Реестр образов:** GitHub Container Registry (GHCR).
*   **Инфраструктура:** VPS (Ubuntu 24.04 LTS) на хостинге Beget.

## 🏗 Архитектура и DevOps-процессы

В рамках выполнения задания был реализован полноценный Pipeline, соответствующий современным стандартам DevOps:

### 1. Контейнеризация
Приложение упаковано в легковесный Docker-образ. 
*   Использован кастомный конфиг `nginx.conf` с увеличенными буферами заголовков (`large_client_header_buffers`), что решает проблему ошибки 400 (Request Header Too Large) при работе на localhost.
*   Настроена ротация логов Docker для экономии места на слабом VPS.

### 2. CI/CD Workflow (GitHub Actions)
Пайплайн разбит на логические этапы:
1.  **Build & Push:** При пуше в ветку `main` происходит сборка Docker-образа и его отправка в GHCR.
2.  **Configuration Delivery:** Файл `docker-compose.yml` доставляется на сервер через протокол SCP.
3.  **Automated Deploy:** Через SSH-соединение сервер выполняет команды авторизации в GHCR, загрузки (pull) свежего образа и перезапуска контейнеров без прерывания обслуживания.

### 3. Оптимизация сервера
Для стабильной работы на конфигурации с минимальным объемом RAM:
*   Настроен файл подкачки (**SWAP**) объемом 2GB.
*   Настроена автоматическая очистка старых Docker-образов (`image prune`) после каждого деплоя.

## 📦 Как запустить локально

Для локального запуска проекта вам потребуется установленный Docker и Docker Compose.

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/PieceTheLord/ВАШ_РЕПО.git
    cd ВАШ_РЕПО
    ```
2.  Запустите контейнер:
    ```bash
    docker compose up -d
    ```
3.  Откройте в браузере: [http://localhost](http://localhost)
